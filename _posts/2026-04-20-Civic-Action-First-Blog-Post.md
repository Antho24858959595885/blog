For our community project, my team is working with the Arkansas Civic Action Network. The main idea of our project is to make a website that helps students submit civic action project proposals, log volunteer hours, and receive feedback from some sort of admin or advisor.

So far, I have helped make the schema alongside my team. Even though another team member was assigned to the schema role, I helped think through the bigger picture of how the schema should work and pointed out changes when they needed to be made.

Working on the schema really fleshed out the scope of the project for me. If a student submits a proposal then the system needs to remember who submitted it, what the proposal is for, and whether it has been approved. If an admin gives feedback, then that feedback also needs to be connected to the correct project. Thinking through those connections made the project feel more realistic for me. I also helped design the more creative aspect of the website mockups, trying to align with the schema and the inital website helped me picture how it should display to the users. Making sure the website isn't confusing to the students or admins is a priority as well. 

My plan moving forward is to work on the back-end student-to-admin relationship. I want to help make sure that a student can submit a project proposal and that the admin can  approve or deny and send feedback back to the student. I also want to make sure that the student side and admin side are separated correctly, so students are only seeing what they are supposed to see.

Another thing I want to focus on is getting as much clarification from our client as possible. We need to ask questions and make sure we know what we are building before we get too far into it. I don't want the website as just something that technically works, but something that can be used.
